CLPY.EXCEPTION¶

Function (set-cause ex cause)¶

Set the cause associated with the exception.

Use NIL to clear it.

Function (return-or-raise-python-error v)¶

Return the value if no Python error occurs.

If error-occurred is T, this function will raise an error; otherwise, V is return.

Function (get kw)¶

Return the corresponding Python exception to the keyword KW.

For example, (GET :|IO-ERROR)| will return PyExc_IOError. This function is the reverse of clpy.exception:from.

Function (get-cause ex)¶
Function (from py-exc)¶

Return the corresponding keyword to the Python exception PY-EXC.

Function (raise-generic-or-python-error &key message)¶

Raise a generic error or a Python error.

If error-occurred return NULL, a GENERIC-ERROR will be raised; Otherwise, a Python error will be raised.

Function (set-context ex ctx)¶

Set the context associated with the exception to CTX.

Use NIL to clear it.

Function (new-exception name &key doc base dict)¶

Cresate a new exception class.

BASE and DICT are usually NULL. You can optionally specify a descriptoin for the exception class.
